🔐 Authentication

Overview

The API uses JWT (JSON Web Tokens) for authentication. Include the token in the Authorization header for all protected endpoints.

POST /api/auth/register

Request Example

curl -X POST http://localhost:5000/api/auth/register \ -H "Content-Type: application/json" \ -d '{ "email": "john@example.com", "password": "SecurePass123!", "fullName": "John Doe", "role": "user" }'

Success Response (201)

{ "user": { "_id": "507f1f77bcf86cd799439011", "email": "john@example.com", "fullName": "John Doe", "role": "user", "createdAt": "2024-03-14T10:30:00Z" }, "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
POST /api/auth/login

Request Example

curl -X POST http://localhost:5000/api/auth/login \ -H "Content-Type: application/json" \ -d '{ "email": "john@example.com", "password": "SecurePass123!", "role": "user" }'

🔌 API Endpoints

Chat Endpoints

POST /api/chat/conversations

Create or get existing conversation

Field Type Description
recipientId string ID of the other participant
recipientModel string "User" or "Doctor"
// Frontend usage await getOrCreateConversation({ recipientId: "507f1f77bcf86cd799439011", recipientModel: "User" }).unwrap();
GET /api/chat/conversations

List all conversations for logged-in user

Query Parameters:

Parameter Type Default
page number 1
limit number 10
GET /api/chat/contacts

Get list of available contacts to start conversation with

Query Parameters:

Parameter Type Description
search string Search by name or email (optional)
GET /api/chat/conversations/:conversationId/messages

Retrieve messages from a conversation

Query Parameters:

Parameter Type Description
limit number Max messages to return (default: 20)
before ISO date Load messages before this date (pagination)
POST /api/chat/conversations/:conversationId/messages

Send a message

Field Type Description
content string Message text (required)
{ "content": "Hello! How are you?" }
POST /api/chat/conversations/:conversationId/read

Mark all messages in conversation as read

📄 Pagination

Conversations and messages are paginated for performance. Use the following parameters:

// Request GET /api/chat/conversations?page=2&limit=10 // Response structure { "data": [ /* array of items */ ], "total": 45, "page": 2, "limit": 10, "totalPages": 5 }

Implementation Example

// Frontend with RTK Query const [page, setPage] = useState(1); const { data } = useListConversationsQuery({ page, limit: 10 }); // Load next page const handleLoadMore = () => { setPage(prev => prev + 1); };

⚠️ Error Handling

Error Response Format

{ "statusCode": 400, "message": "Validation error", "errors": [ { "field": "email", "message": "Invalid email format" } ] }

HTTP Status Codes

Code Status Meaning Action
200 OK Request successful Continue normally
201 Created Resource created Continue normally
400 Bad Request Invalid input Check request body and parameters
401 Unauthorized Invalid/missing token Re-authenticate, refresh token
403 Forbidden Insufficient permissions Check user role and rights
404 Not Found Resource doesn't exist Verify IDs and resource existence
409 Conflict Resource already exists Handle duplicate creation
500 Server Error Internal server error Retry or contact support

Frontend Error Handling Example

const handleLogin = async (credentials) => { try { const response = await login(credentials).unwrap(); // Success } catch (error) { if (error.status === 401) { // Unauthorized - show login prompt } else if (error.status === 400) { // Validation error console.log(error.data.errors); } else { // Generic error showErrorMessage('An error occurred'); } } };

⚡ WebSocket Events

Emitting Events (Client → Server)

// Connect with token const socket = io('http://localhost:5000', { auth: { token: accessToken } }); // Send message (also sent via HTTP) socket.emit('chat:message', { conversationId: '...', content: 'Hello!' }); // Emit typing status socket.emit('chat:typing', { conversationId: '...' });

Receiving Events (Server → Client)

// Listen for new messages socket.on('chat:message', (message) => { // Update chat UI // message: { _id, content, senderId, createdAt, ... } }); // Listen for read receipts socket.on('chat:read', (data) => { // data: { conversationId, messageId, readAt } }); // Listen for user online/offline socket.on('user:online', (data) => { // data: { userId, status: 'online' } }); socket.on('user:offline', (data) => { // data: { userId, status: 'offline' } });

⏱️ Rate Limiting & Best Practices

⚠️ Rate Limit Guidelines

API may implement rate limiting in production. Implement exponential backoff when retrying failed requests.

Request Best Practices

  • Use pagination for large datasets
  • Cache data client-side when appropriate
  • Batch operations when possible
  • Implement proper error handling and retries
  • Use WebSocket for real-time events instead of polling
  • Validate input before sending
  • Keep tokens secure (never expose in logs)

Frontend Optimization

💻 Complete Code Examples

Frontend: Login Flow

import { useLoginMutation } from '@/redux/features/auth/authApi'; import { useDispatch } from 'react-redux'; import { setCredentials } from '@/redux/features/auth/authSlice'; export default function LoginPage() { const [login, { isLoading }] = useLoginMutation(); const dispatch = useDispatch(); const handleSubmit = async (e) => { e.preventDefault(); try { const response = await login({ email: 'user@example.com', password: 'password', role: 'user' }).unwrap(); dispatch(setCredentials({ user: response.user, accessToken: response.accessToken, refreshToken: response.refreshToken })); router.push('/inbox'); } catch (error) { console.error('Login failed:', error.data?.message); } }; return ( <form onSubmit={handleSubmit}> {/* form fields */} </form> ); }

Frontend: Send Message

const [sendMessage, { isLoading }] = useSendMessageMutation(); const handleSend = async () => { try { await sendMessage({ conversationId: activeConversationId, content: messageText }).unwrap(); setMessageText(''); } catch (error) { showError('Failed to send message'); } };

Backend: Custom Route Handler

// routes/chat.routes.ts export const chatRoutes = Router(); chatRoutes.post( '/conversations', authMiddleware, restrictTo('user', 'sender'), validate(createConversationSchema), ChatController.getOrCreateConversation ); // Controller method static async getOrCreateConversation(req, res, next) { try { const conversation = await ChatService.getOrCreateConversation( req.user, req.body ); res.status(201).json({ conversationId: conversation._id }); } catch (error) { next(error); } }
