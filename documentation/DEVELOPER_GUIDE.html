🔧 Backend Architecture Deep Dive

Request Flow

Request → Express Middleware → Route Handler → Controller → Service → Repository → Database ↓ Response Formatter ↓ JSON Response

Key Components

1. Controllers (Controllers/)

Handle HTTP requests and format responses

// Example: chat.controller.ts export default class ChatController { static async getOrCreateConversation(req, res, next) { try { const conversation = await ChatService.getOrCreateConversation( req.user, req.body ); res.status(201).json({ conversationId: conversation._id }); } catch (error) { next(error); } } }

2. Services (Services/)

Contain business logic and orchestration

// Example: chat.service.ts export default class ChatService { static async getOrCreateConversation(user, { recipientId }) { // Validate user has access // Check if conversation exists // Create if doesn't exist // Format and return data } }

3. Repositories (Repositories/)

Data access layer - interact with MongoDB

// Example: chat.repository.ts export default class ChatRepository { static async findConversation(userId, senderId) { return await ChatConversationModel.findOne({ $or: [ { userId, senderId }, { userId: senderId, senderId: userId } ] }).populate(['userId', 'senderId']); } }

4. Models (Models/)

Mongoose schemas and database models

const ChatConversationSchema = new Schema({ userId: { type: Schema.Types.ObjectId, ref: 'Doctor' }, senderId: { type: Schema.Types.ObjectId, ref: 'User' }, initiatedBy: String, lastMessage: String, lastMessageTime: Date, createdAt: { type: Date, default: Date.now } }); // Create unique index ChatConversationSchema.index({ userId: 1, senderId: 1 }, { unique: true });

5. Middleware (Middlewares/)

Auth Middleware

Verifies JWT token from request headers. Extracts user info and attaches to request object.

authMiddleware
Validation Middleware

Validates request body against Joi schemas. Returns 400 if validation fails.

validate(schema)
Error Middleware

Global error handler. Formats errors and sends appropriate HTTP status codes.

errorMiddleware

Backend receives and saves POST /api/chat/conversations/:id/messages → ChatController.sendMessage() → ChatService.sendMessage() → ChatRepository.createMessage() → Emit Socket event // 3. Authentication Flow

// Register/Login 1. User submits credentials 2. Backend validates email 3. (Login: verify password) 4. Generate JWT tokens 5. Return tokens to client 6. Redux stores tokens 7. Subsequent requests include token in header 8. Auth middleware verifies token 9. If valid, attach user to request 10. If invalid, return 401

✅ Best Practices

Backend Development

🔐 Security
  • Never log sensitive data
  • Validate all inputs
  • Use environment variables
  • Implement rate limiting
  • Hash passwords with bcrypt
  • Use HTTPS in production
📊 Database
  • Create indexes for queries
  • Use pagination
  • Avoid N+1 queries
  • Use populate wisely
  • Archive old data
  • Regular backups
🏗️ Architecture
  • Follow MVC pattern
  • DRY principle
  • SOLID principles
  • Error handling layer
  • Middleware chain
  • Type safety (TypeScript)

Frontend Development

⚛️ React
  • Use functional components
  • Memoize when needed
  • Keep components small
  • Extract custom hooks
  • Prop drilling prevention
  • Key prop in lists
🗂️ State Management
  • Use Redux for global state
  • Local state for UI
  • RTK Query for data fetching
  • Avoid prop drilling
  • Selector memoization
  • Normalize state shape
🎨 UI/UX
  • Responsive design first
  • Loading states
  • Error boundaries
  • Accessibility (a11y)
  • Consistent styling
  • User feedback
⚠️ Common Mistakes to Avoid:
  • Storing sensitive data in localStorage
  • Making API calls in render
  • Creating new objects in selectors
  • Not handling loading/error states
  • Hardcoding API URLs
  • Missing null/undefined checks

🔍 Troubleshooting Guide

Common Issues & Solutions

❌ CORS Error

Problem: Cross-Origin Request Blocked
// Solution: Enable CORS in backend (server.ts) const corsOptions = { origin: process.env.FRONTEND_URL || 'http://localhost:3000', credentials: true }; app.use(cors(corsOptions));

❌ Auth Token Not Persisting

Problem: User gets logged out on page refresh
// Solution: Persist Redux store import { persistor } from '@/redux/store'; <PersistGate loading={null} persistor={persistor}> <App /> </PersistGate>

❌ Socket.io Not Connecting

Problem: WebSocket connection fails Checklist:
  • Backend socket.ts is running
  • Socket URL is correct
  • CORS enabled for WebSocket
  • JWT token is valid
  • Check browser console for errors

❌ MongoDB Connection Error

Problem: Cannot connect to database
// Check: 1. MONGODB_URI is correct format 2. Network access enabled in MongoDB Atlas 3. IP whitelist includes your IP 4. Database name exists 5. Credentials are correct

❌ Messages Not Sending

Troubleshooting Steps:
1. Check network tab for POST request 2. Verify response status code 3. Check Redux store for message 4. Verify conversationId is valid 5. Check backend logs for errors 6. Verify token hasn't expired

❌ Slow Queries

Solution: Add Database Indexes
db.conversations.createIndex({ userId: 1, senderId: 1 }) db.messages.createIndex({ conversationId: 1 }) db.messages.createIndex({ createdAt: -1 }) db.users.createIndex({ email: 1 })

Debug Mode

// Backend: Add logging console.log('Request:', req.method, req.path); console.log('User:', req.user._id); console.log('Body:', req.body); // Frontend: Redux DevTools // Install Redux DevTools browser extension // Open DevTools → Redux tab // Track state changes and actions

Performance Optimization

Area Issue Solution

Frontend Large bundle size Code splitting, lazy loading, tree shaking
Backend Slow API responses Add database indexes, pagination, caching
Backend Memory leaks Close connections, clean up listeners
Database Slow queries Indexes, $lookup optimization, aggregation
